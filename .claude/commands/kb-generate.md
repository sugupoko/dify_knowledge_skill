# Skill: kb-generate（ナレッジデータ生成）

`/kb-generate [project_path]` で実行。
designの仕様（spec.md）に従って、ドキュメントから実際のFAQ / ナレッジデータを生成する。

## いつ使うか
- /kb-design で spec.md が生成された後
- 「設計は決まった。実際のFAQデータを作る」時
- ドキュメントそのまま投入の場合はスキップ

---

## Phase 1: 生成方針の確認

### 1.1 spec.md から仕様を読み込む

```
確認項目:
  □ 投入方式: [FAQ化 / ドキュメントそのまま / 混合]
  □ 生成パターン: [A（同時生成）/ B（2ステップ）/ C（既存FAQ変換）]
  □ 対象ドキュメント: [ファイル名のリスト]
  □ カテゴリ設定: [カテゴリのリスト]
  □ 推定件数: [N件]
  □ ナレッジベース構成: [KB名と対応ドキュメント]
```

### 1.2 ドキュメントそのまま投入の場合

```
FAQ生成は不要。以下を行う:
  1. 対象ドキュメントを v{N}/knowledge/ にコピー
  2. チャンキング設計に従ってセクション分割が必要か確認
  3. 必要であればセクション単位に分割したファイルを生成
  → Phase 3 に進む
```

---

## Phase 2: FAQ の生成

### 2.1 パターン別の実行

reference/prompt_templates.md のプロンプトを使い、対象ドキュメントからFAQを生成する。

```
■ 出力先: v{N}/knowledge/faq_draft.csv（TSV形式）

■ 出力フォーマット:
  No | カテゴリ | 質問 | 回答 | 条件/例外 | 次の行動 | エスカレーション | 参照元 | 言い換えキーワード
```

#### パターンA: 同時生成（ドキュメントが短い場合）

```
手順:
  1. 対象ドキュメントを読み込む
  2. prompt_templates.md のパターンAプロンプトを適用
  3. 出力を faq_draft.csv に保存
```

#### パターンB: 2ステップ生成（ドキュメントが長い場合）

```
手順:
  1. 対象ドキュメントをチャンク（章・セクション単位）に分割
  2. 各チャンクに対して:
     Step 1: 質問を生成（5-10件/チャンク）
     Step 2: 各質問に対する回答を生成
  3. 全チャンクの結果を結合して faq_draft.csv に保存

注意:
  - チャンクが大きすぎると品質が落ちる（1章を丸ごと投げない）
  - 章の中のセクション単位で分割するのが目安
```

#### パターンC: 既存FAQ変換

```
手順:
  1. 既存FAQを読み込む
  2. prompt_templates.md のパターンCプロンプトで変換
     - 1FAQ1テーマに分割
     - 結論先出しに書き換え
     - 条件・例外を分離
     - 言い換え・別名を追加
  3. 不足テーマを特定
     - 対象ドキュメントにあるが既存FAQにないトピック
     - 問い合わせログにあるが既存FAQにない質問
  4. 不足分をパターンA or B で追加生成
  5. 結合して faq_draft.csv に保存
```

### 2.2 ナレッジベースが複数ある場合

```
KB別にファイルを分ける:
  v{N}/knowledge/
    ├── kb1_faq_draft.csv   ← KB1用
    ├── kb2_faq_draft.csv   ← KB2用
    └── kb3_faq_draft.csv   ← KB3用

各KBの対象ドキュメントは spec.md に定義されている。
```

## Phase 3: 品質チェック

### 3.1 faq_design_guide.md のチェックリスト

```
□ 1FAQ1テーマになっているか
□ 質問文がユーザーの言い方に近いか
□ 回答の最初の1-2文で結論が分かるか
□ 条件と例外が分かれているか
□ 次の行動が書かれているか
□ エスカレーション条件があるか
□ 参照元が記載されているか
□ 言い換え・別名があるか
```

### 3.2 数量チェック

```
□ 推定件数と実際の生成件数が大きく乖離していないか
  - 推定: [spec.md の値]件
  - 実際: [生成した件数]件
  - 乖離率: [%]
  - 乖離が20%以上の場合、原因を記録

□ カテゴリ別の偏りがないか
  - 特定カテゴリに集中していないか
  - カバレッジが低いカテゴリがないか
```

### 3.3 サンプリングチェック

```
各カテゴリから2-3件をランダムに選び、以下を確認:
  □ 回答が元ドキュメントの内容と一致しているか（ハルシネーションがないか）
  □ 条件・例外が元ドキュメントに存在する内容か
  □ 参照元が実際に存在するか（章番号・セクション名が正しいか）
```

## Phase 4: レビュー用出力

### 4.1 レビューサマリーの提示

```
以下をユーザーに提示する:

■ 生成結果
  - 合計: [N]件
  - カテゴリ別内訳: [テーブル]
  - 生成パターン: [A / B / C]
  - 対象ドキュメント: [ファイル名]

■ 品質チェック結果
  - チェックリスト: [全項目OK / NG項目あり]
  - 数量チェック: [推定 vs 実際]
  - サンプリング: [確認した件数 / 問題の有無]

■ 確認のお願い
  「v{N}/knowledge/faq_draft.csv を確認してください。
   修正があれば教えてください。
   問題なければ /kb-build でDifyに投入します。」
```

### 4.2 修正対応

```
ユーザーから修正指示があった場合:
  1. faq_draft.csv を更新
  2. 修正内容を記録
  3. 再度品質チェック
  4. 再度レビュー提示
```

---

## Phase 5: 出力

**以下を保存すること:**

```
v{N}/knowledge/
  ├── faq_draft.csv          ← 生成したFAQ（メインの成果物）
  ├── kb1_faq_draft.csv      ← KB別ファイル（複数KBの場合）
  └── generation_log.md      ← 生成ログ（パターン、件数、品質チェック結果）
```

**generation_log.md のフォーマット:**

```markdown
## FAQ生成ログ

### 生成設定
- パターン: [A / B / C]
- 対象ドキュメント: [リスト]
- 生成日: [日付]

### 生成結果
| カテゴリ | 件数 |
|---------|------|
| ... | ... |
| **合計** | **N件** |

### 品質チェック
| チェック項目 | 結果 |
|-------------|------|
| 1FAQ1テーマ | [OK/NG] |
| 結論先出し | [OK/NG] |
| ... | ... |

### サンプリング結果
| FAQ No | カテゴリ | 確認結果 | 備考 |
|--------|---------|---------|------|
| ... | ... | ... | ... |

### レビュー状況
- [レビュー待ち / レビュー完了 / 修正済み]
```

## 状態管理

### 読み込み
- `.kb_state.yaml` の `design` セクションを読み込む
- spec.md の FAQ生成仕様を参照する

### 書き込み
- 実行完了時に `.kb_state.yaml` の `generate` セクションを書き込む
- フィールド:
  ```yaml
  generate:
    status: completed
    pattern: "B"
    total_faqs: 70
    categories:
      - name: "カテゴリ名"
        count: 10
    quality_check: "all_passed"
    review_status: "pending"  # pending / reviewed / revised
    output_files:
      - "v1/knowledge/faq_draft.csv"
  ```
