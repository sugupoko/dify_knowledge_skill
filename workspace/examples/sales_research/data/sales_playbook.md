# 営業プレイブック — TechFlow株式会社 営業チーム向け

最終更新: 2026年3月
対象: 営業チーム全員（特にAE・SDR）

## 第1章: 提案時のポイント

### 1.1 提案の基本フレーム「3つのWHY」

提案の冒頭で必ず以下を確認・言語化すること。

```
WHY NOW（なぜ今か）
  → 「今、これを解決しないとどんな機会損失・リスクがあるか」
  → 例: 「Q2に新規事業立ち上げがあり、それまでに営業基盤を整える必要がある」

WHY US（なぜTechFlowか）
  → 「競合との差別化ポイントを顧客の言葉で言えるか」
  → 例: 「国内中堅企業での導入実績と、専任CSによる伴走支援」

WHY CHANGE（なぜ現状を変えるか）
  → 「現状のままだと何が起きるか」
  → 例: 「Excelでの管理は情報が属人化し、担当者が変わるたびに営業情報が失われる」
```

### 1.2 業種別の刺さるメッセージ

| 業種 | 主な課題 | TechFlowで刺すポイント |
|------|---------|---------------------|
| 製造業 | 代理店・販売店の管理が属人化 | CRM Proの代理店管理機能。活動ログの自動記録 |
| IT/SaaS | MRR・チャーン管理が不十分 | TechFlow CS のヘルスチェック・チャーン予測 |
| 商社・卸売 | 案件数が多くパイプラインが見えない | CRM ProのパイプラインUI・売上予測ダッシュボード |
| 不動産 | 追客・フォローのタイミング管理が難しい | MAのリードスコアリング・自動メールシーケンス |
| 人材・採用 | 複数の候補者・求人を同時管理 | CRM Proのカスタムオブジェクト（候補者・求人の管理） |

### 1.3 意思決定者別のアプローチ

| 役職 | 関心事 | 使う言葉 |
|------|--------|---------|
| 経営者・役員 | ROI・競合優位・経営可視化 | 「投資回収期間」「営業利益への影響」「経営の意思決定スピード」 |
| 営業部長 | 目標達成率・予測精度・チームの生産性 | 「パイプライン可視化」「売上予測精度」「営業活動の標準化」 |
| IT部門 | セキュリティ・導入負荷・連携 | 「ISMS認証」「API連携」「既存システムとの共存」 |
| 現場営業 | 使いやすさ・入力負荷の削減 | 「モバイル対応」「自動ログ」「入力が最小限」 |
| 財務・経理 | コスト・契約条件 | 「月額固定」「解約のしやすさ」「無駄なく必要な機能だけ」 |

## 第2章: よくある反論と対処法

### 2.1 価格に関する反論

**反論1: 「Salesforceより高い」**
- 状況確認: どのSalesforceプランとの比較か？
- 対処: Professional（$80/ユーザー）との比較ならTechFlowが安い。Essentials（$25）との比較は機能差を説明
- キラー: 「Salesforceの総保有コスト（TCO）には実装コスト・カスタマイズコストが含まれます。TechFlowは導入費用込みで試算すると多くの場合50%安くなります」
- 証拠: 導入費用比較試算表（営業ツールフォルダに格納）

**反論2: 「GENIEEの方が安い」**
- 状況確認: 今後どんな機能を使いたいか？
- 対処: 今の要件だけなら確かに安い。しかし将来MAやCSを使うなら連携コストが発生
- キラー: 「3年間のTCOで比較しますか？GENIEEはSFA単体なので、MAを追加導入すると同じくらいの投資になるケースが多いです」
- 証拠: 3年間TCO試算比較（CFO向け資料フォルダ）

**反論3: 「予算がない」「費用対効果が見えない」**
- 対処: ROIを数値で示す
- フレーム: 「営業1人が1日に使う情報収集・入力作業が何時間か？」→ 削減効果を試算
- 数値例: 「営業10名が1日30分削減 × 250日 = 1,250時間/年。時給3,000円換算で375万円」

### 2.2 機能・品質に関する反論

**反論4: 「AI機能が弱い」**
- 認める: 「Einstein AIやHubSpotのAI機能と比べると、現時点では機能が限定的です」
- リフレーム: 「AIに頼りすぎると、入力データの品質次第で予測が外れることも多いです。まず正確なデータを蓄積する基盤を作ることが先決で、その点でTechFlowは強みがあります」
- 将来性: 「2026年Q3にAI機能の大幅アップデートを予定しています（ロードマップ参照）」

**反論5: 「グローバル対応が弱い」**
- 確認: 「現在、または近い将来のグローバル展開の可能性はどのくらいありますか？」
- 対処: グローバル展開が確実なら正直に「Salesforceをお勧めします」
- 残存機会: 「日本法人の営業管理にTechFlowを使いつつ、グローバルはSalesforceという分離運用のお客様もいます」

**反論6: 「カスタマイズが難しい」**
- 確認: 「具体的にどんなカスタマイズが必要ですか？」
- 対処: 90%のケースはノーコードで実現可能
- エスカレーション基準: カスタムオブジェクトの複雑な連携・API開発が必要な場合は、技術営業（SE）を同席させる

### 2.3 比較・検討段階での反論

**反論7: 「今すぐ決められない。もう少し検討したい」**
- 状況確認: 「検討に必要な情報が不足している部分はありますか？」
- 行動提案: 「無料トライアル（30日間）を使って、実際の商談データで試してみてはいかがでしょうか？」
- 締め切りの設定: 「2週間後にフィードバックをいただけますか？ご不明点があれば技術担当を交えて詳しくご説明します」

**反論8: 「他のベンダーも検討している」**
- 受け入れる: 「もちろんです。どちらのベンダーと比較されていますか？」
- 差別化の確認: 「比較の中で最も重視されている軸はどれですか？」（価格・機能・サポート等）
- 対処: 重視軸に合わせてTechFlowの強みを再提示

## 第3章: 見積もりルール

### 3.1 見積もりの基本原則

1. **標準価格を起点にする**: 割引は理由なく提示しない
2. **年払い優先**: 月払いより年払いを先に提示（年払いは月払い比10%割引）
3. **セット提案を先にする**: 単製品より複数製品のセット提案を優先（成約率が高い）
4. **見積もり有効期限**: 発行日から30日間

### 3.2 標準的な見積もり構成

```
【見積もり例: 営業チーム20名への CRM Pro 提案】

製品: TechFlow CRM Pro Growth
ユーザー数: 20名
プラン: 月払い
単価: 9,600円/ユーザー/月
小計: 192,000円/月

初期費用: 
  オンボーディング支援: 150,000円（一括）
  データ移行支援（オプション）: 200,000円〜（規模による）

年間総額（月払いの場合）: 192,000円 × 12 = 2,304,000円 + 初期費用

【年払い提案の場合】
単価: 8,640円/ユーザー/月（10%割引）
年間: 8,640円 × 20名 × 12ヶ月 = 2,073,600円
節約額: 約230,000円
```

### 3.3 セット割引の適用ルール

- セット割引は正式な見積書に適用（口頭では言及しない）
- 割引率は標準表の通り（独自割引は不可）
- Enterprise案件（月額500万円以上）は上長承認後に個別設定可能

## 第4章: 値引き基準

### 4.1 営業担当者の権限

| 割引率 | 承認者 | 条件 |
|--------|--------|------|
| 0〜10% | 営業担当者の判断 | 年払い選択、複数年契約、参照事例の提供承諾 |
| 11〜20% | 営業マネージャー承認 | 競合他社との価格差が明確、大型案件（月額200万円以上） |
| 21〜30% | VP of Sales承認 | 戦略的アカウント、上場企業・業界リーダー、プレスリリース承諾 |
| 31%以上 | 禁止（例外は役員案件のみ） | — |

### 4.2 値引きの正当化ルール

値引きを提示する場合、必ず以下のいずれかの理由を明示する:

1. **年払い・複数年契約**: 「年払い選択のため10%割引」
2. **複数製品セット**: セット割引表を適用
3. **参照事例提供**: 「導入事例のインタビュー・ロゴ掲載承諾の場合5%割引」
4. **競合見積もりへの対抗**: 競合見積もりコピーを上長に提出の上、承認を得る
5. **初回導入記念**（新規顧客向け）: 「初年度のみ10%割引キャンペーン適用」

### 4.3 値引きをしてはいけないケース

- 「なんとかならない？」という理由のない値引き要求
- 上限を超えた値引きの事前約束（後で否認が必要になる）
- 単年・月払いのままでの値引き（年払いへの誘導を先に行う）

## 第5章: エスカレーション基準

### 5.1 技術営業（SE）のアサイン基準

以下のいずれかに該当する場合は、商談にSEを同席させる:

- [ ] API連携・システム連携の要件がある
- [ ] セキュリティ要件の詳細確認が必要（ISMS・SOC2・ペネトレーションテスト等）
- [ ] カスタマイズ要件が複雑（カスタムオブジェクトの複雑な連携）
- [ ] データ移行の規模が大きい（100,000件以上のデータ移行）
- [ ] Enterprise提案でRFP（提案依頼書）への回答が必要

### 5.2 上長（マネージャー）へのエスカレーション基準

- [ ] 競合対抗値引きが必要で、11%以上の値引きが必要
- [ ] 顧客からのクレーム・不満が発生している
- [ ] 商談が3ヶ月以上停滞しており、打開策が見えない
- [ ] 特殊な契約条件（秘密保持・SLAの個別交渉）が求められている
- [ ] 公共機関・官公庁への提案（入札対応が必要）

### 5.3 VP of Sales へのエスカレーション基準

- [ ] 月額300万円以上の大型案件
- [ ] 21%以上の値引きが必要
- [ ] 競合（Salesforce等）との間でエグゼクティブレベルの関係構築が必要
- [ ] M&AやIPO関連で経営者同士の面会が有効なケース

### 5.4 法務・コンプライアンスへのエスカレーション基準

- [ ] 個人情報処理委託契約の特別条項が求められる
- [ ] 顧客からの独自契約書（カスタム利用規約）への同意を求められる
- [ ] 海外データ移転に関する確認が必要
- [ ] 反社会的勢力チェックが必要と判断されるケース
