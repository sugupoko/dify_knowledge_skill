# 社内IT利用マニュアル（サンプル）

## 1. VPN接続

### 1.1 接続方法
1. GlobalProtect アプリを起動
2. ポータルアドレスに `vpn.example.co.jp` を入力
3. 社員ID とパスワードでログイン
4. 「接続」をクリック
5. ステータスが「接続済み」になれば完了

### 1.2 よくあるトラブル

#### 「認証に失敗しました」
- パスワードの有効期限切れ → パスワードリセット（2.2参照）
- アカウントロック → ヘルプデスクに連絡（2.3参照）
- 多要素認証の設定不備 → 管理者に連絡

#### 「接続はできるがファイルサーバーにアクセスできない」
- スプリットトンネル設定の場合、一部の社内リソースに直接アクセスできないことがある
- フルトンネルモードに変更: GlobalProtect の設定 → 接続モード → 「フルトンネル」を選択

#### 「接続が遅い」
- フルトンネルの場合、全通信がVPN経由になるため遅くなることがある
- 対処: 大容量ファイルのダウンロードはVPN切断後に行う
- 改善しない場合: 接続先ゲートウェイの変更を試す（設定 → ゲートウェイ → 別リージョンを選択）

### 1.3 VPNアカウントの申請
- 対象: 正社員・契約社員・派遣社員（派遣元の契約による）
- 申請: 社内ポータル → IT申請 → 「VPNアカウント発行」
- 承認フロー: 申請者 → 直属上長 → 情報セキュリティ部門
- 所要期間: 通常3営業日

## 2. アカウント・パスワード管理

### 2.1 パスワードポリシー
- 最低12文字以上
- 大文字・小文字・数字・記号のうち3種類以上
- 過去12回分のパスワードは再利用不可
- 有効期限: 90日（期限切れ前にメールで通知）
- 5回連続で間違えるとアカウントロック

### 2.2 パスワードリセット
1. セルフサービスポータル `https://password.example.co.jp` にアクセス
2. 社員ID を入力
3. 登録済みの個人メールアドレスに認証コードが届く
4. 認証コードを入力
5. 新しいパスワードを設定（ポリシーに従うこと）

**セルフサービスが使えない場合:**
- 個人メールアドレスが未登録 → ヘルプデスクに電話（内線9999）
- 本人確認: 社員番号 + 生年月日 + 所属部署

### 2.3 アカウントロック解除
- 自動解除: 30分後
- 即時解除: ヘルプデスクに連絡（内線9999）
  - 本人確認が必要（社員番号 + 生年月日）

### 2.4 異動時の対応
- メールアドレス: 変更なし（全社共通ドメイン）
- 部署表示: 人事システムと連動して自動更新（異動日翌日）
- アクセス権: 旧部署の権限は異動日に自動削除。新部署の権限は上長が申請。
- **注意**: 旧部署の共有フォルダにアクセスが残っている場合は、情報セキュリティ部門に報告すること

### 2.5 退職時のアカウント
- 最終出社日にアカウント停止（メール送受信不可）
- 退職日に完全削除
- 引き継ぎ用メールが必要な場合: 上長が「メール転送設定」を申請（最長30日間）

## 3. PC管理

### 3.1 標準PC仕様
- ノートPC: Lenovo ThinkPad T14s / Dell Latitude 5440
- OS: Windows 11 Pro
- メモリ: 16GB
- ストレージ: 512GB SSD
- 更新サイクル: 4年

### 3.2 新規PCセットアップ手順
1. 電源を入れ、初期設定ウィザードに従う
2. Wi-Fi に接続（社内: `CORP-WIFI` / パスワードは貸与時の書類に記載）
3. Microsoft 365 にサインイン（社員ID@example.co.jp）
4. Windows Update を実行（再起動が複数回必要）
5. GlobalProtect（VPN）をインストール → 1.1 参照
6. プリンタードライバをインストール → 6.1 参照
7. 必要なソフトウェアの申請 → 4.1 参照

### 3.3 代替機の貸出
- 故障・修理中に代替機を利用可能
- 申請: ヘルプデスクに連絡（内線9999）
- 在庫状況により即日〜翌営業日
- 貸出期間: 最長2週間

### 3.4 外部モニター接続
- USB-C / HDMI / DisplayPort に対応
- 映らない場合:
  1. ケーブルを抜き差し
  2. `Windows + P` で表示モードを「複製」or「拡張」に変更
  3. ドッキングステーション経由の場合、ドッキングステーションのファームウェアを確認

### 3.5 バッテリー交換
- 対象: 使用開始から2年以上 かつ バッテリー持続時間が著しく低下
- 申請: 社内ポータル → IT申請 → 「PC修理・交換」
- 対応: バッテリー交換 or PC更新（使用期間3年以上の場合は更新推奨）

## 4. ソフトウェア管理

### 4.1 ソフトウェア申請
- 全てのソフトウェアは申請制
- 申請: 社内ポータル → IT申請 → 「ソフトウェアインストール」
- 承認: 上長 → IT部門（セキュリティチェック）
- 所要期間: 許可リスト掲載済み=1営業日、未掲載=5〜10営業日（セキュリティ審査）

### 4.2 管理者権限
- 原則、一般ユーザーには付与しない
- 開発業務等で必要な場合:
  - 申請: 社内ポータル → IT申請 → 「管理者権限付与」
  - 承認: 上長 → 部長 → 情報セキュリティ部門
  - 付与条件: セキュリティ研修を受講済みであること
  - 期間: 最長1年（更新申請が必要）

### 4.3 ライセンス管理
- Microsoft 365: 全社員に付与（E3プラン）
- Adobe Acrobat Pro: 申請制（部門予算）
- Zoom: 基本は無料プラン。有料プランは部門長承認
- Slack: 有料プランへの追加は情報システム部に依頼
- 開発ツール（Visual Studio、JetBrains等）: 開発部門は標準配布、他部門は申請制

## 5. メール・コミュニケーション

### 5.1 メールボックス容量
- 上限: 50GB
- 容量不足時:
  1. 不要メールの削除（特に添付ファイル付き）
  2. 「削除済みアイテム」の完全削除
  3. 1年以上前のメールをアーカイブ
  4. 上記で解決しない場合 → ヘルプデスクに連絡

### 5.2 共有メールボックス
- 申請: 社内ポータル → IT申請 → 「共有メールボックス作成」
- 承認: 部門長
- 所要期間: 3営業日
- メンバー追加・削除: 管理者が社内ポータルから操作

### 5.3 メーリングリスト
- メンバー追加・削除: リストの管理者が操作
- 管理者が不明な場合: ヘルプデスクに問い合わせ
- 新規作成: 社内ポータル → IT申請 → 「メーリングリスト作成」

### 5.4 メール受信トラブル
- 社外からのメールが届かない場合:
  1. 迷惑メールフォルダを確認
  2. 送信者のドメインがブロックリストに入っていないか確認
  3. 解決しない場合 → ヘルプデスクに連絡（送信者のメールアドレスと送信日時を添えること）

### 5.5 退職者のメール引き継ぎ
- 上長が「メール転送設定」を申請
- 転送先: 後任者のメールアドレス
- 期間: 最長30日間
- 転送開始: 最終出社日の翌日
- 注意: 転送されるのは新規受信メールのみ。過去メールの閲覧が必要な場合は別途相談

## 6. プリンター・複合機

### 6.1 プリンタードライバのインストール
1. 社内ポータル → ダウンロード → 「プリンタードライバ」
2. フロア別のドライバをダウンロード
3. インストーラーを実行
4. テスト印刷で確認

### 6.2 紙詰まり
1. 前面カバーを開ける
2. 詰まった用紙をゆっくり引き出す（無理に引っ張らない）
3. 紙片が残っていないか確認
4. カバーを閉める
5. エラーが消えない場合 → 電源を入れ直す
6. それでも消えない場合 → ヘルプデスクに連絡

### 6.3 スキャン設定
- 自分のメールに送る場合:
  1. 複合機のタッチパネルで「スキャン」を選択
  2. 「メール送信」を選択
  3. 宛先に自分のメールアドレスを入力（アドレス帳から選択も可）
  4. 「送信」をタップ
- アドレス帳の登録・変更: ヘルプデスクに依頼

## 7. セキュリティルール

### 7.1 フィッシングメールの対応
1. リンクはクリックしない
2. 添付ファイルは開かない
3. メールを「フィッシング報告」ボタンで報告（Outlookツールバーにあり）
4. 不安な場合はヘルプデスクに転送（helpdesk@example.co.jp）

**既にリンクをクリックしてしまった場合:**
1. すぐにネットワークを切断（Wi-Fiオフ / LANケーブルを抜く）
2. ヘルプデスクに電話（内線9999）— 緊急扱い
3. パスワードを入力してしまった場合は、別の端末から即座にパスワード変更

### 7.2 データ持ち出し
- USBメモリ: 原則禁止。例外は情報セキュリティ部門の事前承認が必要
- クラウドストレージ: 承認済みサービス（OneDrive、SharePoint）のみ使用可
- メール添付: 社外秘以上は暗号化必須（パスワード別送）
- 印刷物: 持ち出す場合は上長承認

### 7.3 インシデント報告
- 情報漏洩（疑い含む）が発生した場合:
  1. 即座に上長に報告
  2. 上長が情報セキュリティ部門に連絡
  3. 指示があるまで、該当データの操作を停止
  4. 経緯を時系列で記録

### 7.4 BYOD（個人端末の業務利用）
- 個人スマホへのTeams/Outlookインストール: 可（MDM登録が必要）
- MDM登録方法: 社内ポータル → IT申請 → 「BYOD登録」
- 注意: MDM登録により、紛失時にリモートワイプ（業務データ削除）が実行される場合がある

### 7.5 セキュリティ研修
- 全社員: 年1回（eラーニング、毎年4月）
- 新入社員: 入社時に必須（対面 or オンライン）
- 管理者権限保有者: 年2回（4月・10月）
- 未受講の場合: 一部システムのアクセスが制限される

### 8.1 備品申請
- 申請: 社内ポータル → 総務申請 → 「備品購入」
- 承認: 上長（5万円未満） / 部長（5万円以上）

## 9. クラウドサービス・SaaS

### 9.1 新規SaaS導入
- セキュリティ審査が必要
- 申請: 社内ポータル → IT申請 → 「新規クラウドサービス利用申請」
- 審査項目: データ保管場所、暗号化、SLA、退会時のデータ削除方針
- 所要期間: 2〜4週間
- 承認: IT部門 → 情報セキュリティ部門 → CIO

### 9.2 外部ゲスト招待
- Microsoft Teams: ゲストアクセス可（チームオーナーが招待）
- Notion: ゲスト招待は情報セキュリティ部門の事前承認が必要
- Google Workspace: 外部共有は原則禁止。例外は部長承認

### 9.3 ストレージ容量
- OneDrive: 1TB/人
- SharePoint: チームサイトごとに25TB
- Google Workspace: 部門プールで管理。追加はIT部門に依頼

## 10. 障害発生時の対応

### 10.1 社内システム障害
- ステータスページ: `https://status.example.co.jp` で確認
- 広範囲の障害: 全社メール + Teamsで通知
- 復旧見込みが不明な場合: ヘルプデスクに確認（ただし障害中は電話が混雑する）

### 10.2 ランサムウェア感染（疑い）
1. **即座に** LANケーブルを抜く / Wi-Fiを切る
2. PCの電源は**切らない**（証拠保全のため）
3. ヘルプデスクに電話（内線9999）— 最優先対応
4. 周囲の人にも注意喚起
5. 絶対に身代金を支払わない

### 10.3 データ復旧
- 共有フォルダ: 日次バックアップあり。最大30日前まで復旧可能
- 個人PC: OneDriveに同期していれば復旧可能。ローカルのみは不可
- 復旧依頼: ヘルプデスクに連絡（ファイルパス、最終確認日時を添えること）
