# ヒアリングシート（エージェント向け）

エージェントは「自律的にツールを使い、複数ステップで推論する」。
このシートでは、**何を使って・どんなルールで・どこで止まるか**を引き出す。

**使い方**: 業務担当者に記入してもらう。または、このシートをベースにヒアリングを行う。

## ブロック3: 暗黙のNG回答パターン（最重要）

エージェントが誤って行動するリスクを防ぐ。
このブロックが最も重要。「言葉にしていないルール」を必ず引き出す。

```
■ 内容に関する暗黙のNG（絶対に聞く）

Q. 「正確には合っているが、言い方や文脈によってはまずい」回答はありますか？
  例: 「規定上は可能だが、実際には社風的にNGとされている」
  具体例: ____

Q. 過去に「ボット（AI）がこう回答してトラブルになった」ケースはありますか？
  どんなトラブルでしたか？: ____
  なぜ問題でしたか？: ____
  → ボットで同じことが起きないように: ____

Q. 「これは人が言うからOKだが、ボットが言うとおかしい」ケースはありますか？
  例: 「申し訳ありません」などの謝罪表現を使うと責任問題になる
  具体例: ____

■ 行動に関する暗黙のNG

Q. 「エージェントが勝手にやってはいけない」行動はありますか？
  例: 「申請を代わりに送信してしまう」「担当者に無断で転送する」
  NG行動リスト: ____

Q. 「このユーザーにはこの情報を見せてはいけない」という権限制限はありますか？
  例: 「一般社員に役員情報を見せてはいけない」
  権限制限: ____

■ ユーザー対応の暗黙のNG

Q. 「このタイプのユーザーには特別な対応が必要」はありますか？
  例: 「VIP顧客は必ず人が対応する」「クレームは一切ボットで対応しない」
  特別対応が必要なケース: ____

Q. 感情的なメッセージ（怒り、不安、悲しみ）が来た場合、どう対応すべきですか？
  □ 通常通り情報を提供する
  □ 共感を示してから情報を提供する
  □ 必ず人に引き継ぐ
  □ その他: ____
```

## ブロック4: エスカレーション先

エージェントが解決できない場合の引き継ぎ先を定義する。

```

  状況1: ____  → エスカレーション先: ____（メール/電話/Slack: ____）
  状況2: ____  → エスカレーション先: ____
  状況3: ____  → エスカレーション先: ____

Q. エスカレーションの際、どこまでの情報を引き継ぎますか？
  □ 会話履歴全体
  □ 要約のみ
  □ ユーザーの問い合わせ内容のみ
  □ その他: ____

Q. エスカレーション後もボットの会話は続きますか？
  □ 担当者への引き継ぎ後は会話終了
  □ 担当者が対応中もボットがサポート
  □ その他: ____

Q. 24時間・休日の問い合わせはどう扱いますか？
  □ ボットで一次対応、次営業日に人が回答
  □ 緊急連絡先のみ案内
  □ その他: ____
```
