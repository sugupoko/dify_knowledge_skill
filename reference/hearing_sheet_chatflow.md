# ヒアリングシート（チャットフロー向け）

チャットフローは「分類+ルーティング+複数ナレッジベース」の組み合わせで動く。
このシートでは、**どう分類するか・どこにルーティングするか・各経路で何が必要か**を引き出す。

**使い方**: 業務担当者に記入してもらう。または、このシートをベースにヒアリングを行う。

---

```

ヒアリング日: ____
```

## ブロック1: 分類カテゴリの定義

問い合わせをどのカテゴリに分けるかが、チャットフローの設計の核心。

```
Q. 現在、どんな種類の問い合わせがきますか？
  （例: 総務系、IT系、経理系、人事系 / 申請系、調査系、クレーム系）

カテゴリ一覧（お客様の業務に合わせて記入してください）:
  カテゴリ1: ____（例: 総務・庶務）
  カテゴリ2: ____（例: IT・システム）
  カテゴリ3: ____（例: 経理・費用）
  カテゴリ4: ____（例: 人事・労務）
  カテゴリ5: ____（その他: ____）

Q. カテゴリが判断しにくい境界ケースはありますか？
  例: 「経費精算のシステム操作方法」→ 経理系？IT系？
  境界ケース: ____
  その場合はどちらに回しますか？: ____

Q. 全体の中でどのカテゴリが最も多いですか（上位3位まで）？
  1位: ____（全体の約____%）
  2位: ____（全体の約____%）
  3位: ____（全体の約____%）
```

## ブロック2: 条件分岐のルール

同じカテゴリの中でも、条件によって回答が変わる場合。

```
Q. 「Aの場合はこちら、Bの場合はあちら」という分岐はありますか？
  例: 「正社員と契約社員で規定が違う」「金額によって承認フローが違う」

分岐条件の一覧（分かる範囲で記入）:
  分岐1: ____
    条件A: ____  → 回答: ____
    条件B: ____  → 回答: ____

  分岐2: ____
    条件A: ____  → 回答: ____
    条件B: ____  → 回答: ____

Q. 条件を確認するための追加質問が必要な場合はありますか？
  例: 「雇用形態を教えてもらえますか？」と聞いてから案内する
  必要な追加質問: ____

Q. 条件を複数持つケース（AND / OR）はありますか？
  例: 「正社員かつ管理職の場合のみ」
  具体例: ____
```

## ブロック3: ルーティング先の一覧

各カテゴリ・条件の先に何があるか。

```
Q. カテゴリ別に「誰が（どのチームが）回答を担当しますか？」
  カテゴリ1（____）: ____部門 / ____担当 / ____窓口
  カテゴリ2（____）: ____部門 / ____担当 / ____窓口
  カテゴリ3（____）: ____部門 / ____担当 / ____窓口

Q. ボットで自動回答するカテゴリと、必ず人に回すカテゴリを教えてください。
  自動回答OK: ____
  必ず人に回す: ____
  ケースバイケース（どんな条件で分かれるか）: ____

Q. 人に引き継ぐ場合、どのように渡しますか？
  □ 担当者の連絡先（メール/電話）を案内する
  □ Slackチャンネルに転送する
  □ チケットシステムに登録する
  □ その他: ____

Q. 「分からない」と返した後のフォールバック先は？
  □ 総合窓口の連絡先を案内
  □ FAQ一覧ページに誘導
  □ 担当者に転送（メール/チャット）
  □ その他: ____
```

## ブロック4: 暗黙のルール（最重要）

チャットフローで見落としやすい暗黙のルールを引き出す。

```
■ 言ってはいけないこと・やってはいけないこと（絶対に聞く）

Q. ボットが絶対に回答してはいけない内容はありますか？
  □ 個人の給与・評価情報
  □ 競合他社との比較
  □ 未発表の製品・施策
  □ 法的判断・医療的判断
  □ その他: ____

Q. 答えが「場合による」で、ボットが独断で答えてはいけない場合は？
  例: 「例外申請の可否」「特別対応の可否」
  具体例: ____

■ 順序・タイミングの暗黙ルール

Q. 「この情報を先に確認してから回答する」というケースはありますか？
  例: 「本人確認が取れない限り、個人情報に関する回答は控える」
  具体例: ____

Q. 「このカテゴリの前に必ずこちらを案内する」というルールはありますか？
  例: 「ITサポートの前に、まずセルフサービスポータルを案内する」
  具体例: ____

■ NG回答パターン（最重要）

Q. 過去に問題になった回答・クレームになった対応はありますか？
  どんな対応が問題でしたか？: ____
  → ボットで同じことが起きないように、どう対処すべきですか？: ____

Q. 「正確には正しいが、言い方がまずい」という表現のパターンはありますか？
  避けるべき表現: ____
  代わりに使うべき表現: ____
```
